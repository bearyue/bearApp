# BearAsset V1.0 开发计划

> **开发模式**：1 人，业余时间，不限工期
> **原则**：每个阶段产出可运行的增量版本，避免长期看不到成果
> **V1 定位**：个人及家人使用，自部署服务端，非商业化多用户场景

---

## 阶段总览

| 阶段 | 名称 | 预估工期 | 核心产出 |
|---|---|---|---|
| P0 | 项目脚手架 | 1 周 | 工程跑通，空壳 App + 空壳后端 |
| P1 | 用户系统 | 1-2 周 | 管理员建号/登录/JWT/生物识别 |
| P2 | 资产 CRUD | 2-3 周 | 全类型资产的增删改查 + 本地存储 |
| P3 | 首页仪表盘 | 1-2 周 | 净值总览 + 分类卡片 + 日涨跌 |
| P4 | 多币种 + 行情 | 2-3 周 | 汇率同步 + 行情拉取（API/爬虫） |
| P5 | 报表分析 | 2-3 周 | 曲线图/饼图/XIRR/风险分析 |
| P6 | AI Agent | 2-3 周 | AI Gateway + 对话 + 图像识别 + Function Calling |
| P7 | 数据同步与导入导出 | 1-2 周 | 单向备份 + 加密导出导入 |
| P8 | 打磨与发布 | 1-2 周 | Bug 修复 + UI 打磨 + 性能优化 |

**总计预估**：13-21 周（业余时间，弹性节奏）

## P0：项目脚手架（1 周）

### 目标
工程骨架搭建完成，前后端能跑通一个 Hello World 级别的请求链路。

### 任务清单

**Android 端：**
- [ ] 创建 Android 项目，配置 Kotlin + Jetpack Compose
- [ ] 集成 Hilt 依赖注入
- [ ] 集成 Room 数据库（空 DB，定义 Migration 机制）
- [ ] 集成 Retrofit + OkHttp（配置 BaseURL、拦截器、日志）
- [ ] 集成 Navigation Compose，搭建基础导航框架（底部 Tab）
- [ ] 创建基础主题（Material Design 3，工具类简洁风配色）
- [ ] 建立包结构：`ui` / `data` / `domain` / `di` / `network`

**后端：**
- [ ] 创建 Spring Boot 3.x 项目
- [ ] 配置 MyBatis-Plus + PostgreSQL 数据源
- [ ] 配置统一返回格式（Result<T>）和全局异常处理
- [ ] 编写 Dockerfile，本地可 docker-compose 启动
- [ ] 搭建基础包结构：`controller` / `service` / `mapper` / `entity` / `config`

**验收标准**：App 启动后能调通后端的 health check 接口。

## P1：用户系统（1-2 周）

### 目标
完成用户管理、登录、JWT 认证、生物识别快捷登录。管理员创建账号，不开放公开注册。

### 任务清单

**后端：**
- [ ] 建 `user` 表
- [ ] 管理员创建用户接口 `POST /api/admin/user`（密码 BCrypt 加密）
- [ ] 登录接口 `POST /api/auth/login`（返回 JWT Token）
- [ ] Spring Security + JWT 过滤器配置
- [ ] Token 刷新机制（可选，简单方案：长过期时间）
- [ ] 初始化脚本：首次启动自动创建管理员账号

**Android 端：**
- [ ] 登录页 UI（Compose）
- [ ] Token 本地持久化（EncryptedSharedPreferences）
- [ ] OkHttp 拦截器自动附加 Authorization Header
- [ ] 生物识别集成（BiometricPrompt）
  - 首次登录成功后提示开启
  - 后续启动 App 时指纹/面部解锁直接进入
- [ ] 登录状态管理（ViewModel + StateFlow）

**验收标准**：管理员创建账号 → 登录 → 获取 Token → 指纹解锁重新进入。

## P2：资产 CRUD（2-3 周）

### 目标
支持全部资产类型的新增、编辑、删除、列表展示，数据存储在本地 Room 数据库。

### 任务清单

**数据层：**
- [ ] 定义 Room Entity：`AssetEntity`（含所有通用字段 + extra_json）
- [ ] 定义 Room DAO：增删改查、按类型查询、按用户查询
- [ ] 定义 Repository 接口和实现
- [ ] 资产类型枚举定义（Category + SubType）

**UI 层（Compose）：**
- [ ] 资产明细列表页
  - 按大类分组（Liquid/Investment/Restricted/Physical/Liability）
  - 每项显示名称、金额、盈亏
  - 搜索/筛选功能
- [ ] 添加资产页
  - 第一步：选择资产大类 → 子类型
  - 第二步：根据子类型动态展示表单字段
  - 证券类：代码、名称、数量、成本价、当前价
  - 固收类：本金、利率、到期日
  - 房贷类：总额、剩余本金、月供、利率
  - 公积金：基数、比例、当前余额
  - 通用：名称、金额、币种、备注
- [ ] 资产详情/编辑页
- [ ] 删除确认弹窗
- [ ] 公积金月度自动模拟增量逻辑

**验收标准**：能录入各类型资产，列表展示正确，编辑/删除正常。

---

## P3：首页仪表盘（1-2 周）

### 目标
首页展示净值总览，一眼看清家底。

### 任务清单

- [ ] 首页 UI 布局（Compose）
  - 顶部：净值大数字 + 日涨跌额/百分比
  - 中部：总资产 | 总负债 条形展示
  - 下部：分类卡片列表（每类显示小计金额和资产数量）
- [ ] 净值计算逻辑
  - 总资产 = 流动 + 投资 + 受限 + 实体
  - 总负债 = 所有负债项之和
  - 净值 = 总资产 - 总负债
- [ ] 日涨跌计算
  - 对比当日和前一日的投资类资产总市值差值
  - 需要 `net_worth_snapshot` 本地表支持
- [ ] 净值快照定时记录
  - Room 本地表 `NetWorthSnapshotEntity`
  - 每日首次打开 App 时自动保存当天快照
- [ ] 快捷入口：添加资产按钮、AI 对话入口

**验收标准**：首页正确显示各类资产汇总和净值。

## P4：多币种 + 行情数据（2-3 周）

### 目标
港股/美股资产自动折算人民币，证券/基金类资产价格自动更新。

### 任务清单

**后端 - 汇率服务：**
- [ ] 建 `exchange_rate` 表
- [ ] 汇率拉取定时任务（每日一次）
  - 数据源选型：免费汇率 API（如 exchangerate-api.com / 央行数据）
  - 降级：爬虫获取
- [ ] 汇率查询接口 `GET /api/rate/latest`
- [ ] 汇率历史查询接口（用于历史净值回算）

**后端 - 行情服务：**
- [ ] 行情数据表（缓存在后端减少重复请求）
- [ ] 行情拉取逻辑（定时任务，工作日收盘后执行）
  - A 股：数据源选型（待定：免费 API 或爬虫）
  - 港股/美股：数据源选型（待定）
  - 公募基金：天天基金等数据源
- [ ] 行情查询接口 `GET /api/quote/{code}`
- [ ] 批量查询接口 `POST /api/quote/batch`

**Android 端：**
- [ ] 拉取最新汇率，缓存到 Room
- [ ] 资产列表显示原币金额 + 折合人民币金额
- [ ] 拉取行情数据，自动更新证券/基金类资产的当前价
- [ ] 支持手动刷新行情
- [ ] 设置页增加汇率查看/手动刷新入口

**验收标准**：港股/美股资产正确折算人民币；行情数据能自动或手动更新。

## P5：报表分析（2-3 周）

### 目标
完整的数据可视化报表体系。

### 任务清单

**图表库集成：**
- [ ] 选型：MPAndroidChart / Vico / 其他 Compose 友好的图表库
- [ ] 封装通用图表组件

**报表页面：**
- [ ] 净资产曲线
  - 基于 `net_worth_snapshot` 数据
  - 支持切换时间维度（近 1 月 / 近 1 年 / 全部）
  - 拆解：本金投入增量 vs 投资收益增量
- [ ] 资产配置饼图
  - 按大类占比
  - 按子类占比
  - 点击扇区跳转到对应明细
- [ ] 风险分析卡片
  - 权益类 vs 固收类比例
  - 紧急预备金覆盖月数（流动资产 / 用户设定的月支出）
  - 单一资产集中度提醒
- [ ] 投资绩效
  - 单项资产收益率（(当前市值-成本)/成本）
  - XIRR 计算（基于资产变动日志的资金流）
  - 持仓盈亏排行

**数据支撑：**
- [ ] 建 Room 本地表 `asset_change_log`（记录每次资产变动）
- [ ] 净值快照历史查询 DAO

**验收标准**：报表页能正确展示曲线图、饼图、绩效数据。

## P6：AI Agent（2-3 周）

### 目标
接入大模型，实现对话查询、AI 辅助录入、图像识别。

### 任务清单

**后端 - AI Gateway：**
- [ ] AI 模型配置表 `ai_model_config`（支持多模型配置）
- [ ] 统一 AI 请求代理接口 `POST /api/ai/chat`
  - 支持 OpenAI 兼容格式（大多数模型都兼容）
  - 根据配置路由到不同模型
  - API Key 存储在后端，客户端不接触密钥
- [ ] Function Calling 定义
  - `query_assets`：查询资产数据
  - `add_asset`：新增资产
  - `update_asset`：修改资产
  - `get_net_worth`：获取净值
  - `get_asset_allocation`：获取资产配置
- [ ] 系统 Prompt 设计（角色：个人财务助理）
- [ ] 图像识别接口 `POST /api/ai/vision`（转发 Vision API）
- [ ] AI 对话历史表 `ai_chat_history`

**Android 端：**
- [ ] AI 对话页 UI（聊天气泡界面）
  - 支持文字输入
  - 支持拍照/选图发送
  - 流式响应展示（SSE）
- [ ] 确认卡片组件
  - AI 解析结果展示为结构化卡片
  - 用户可修改字段
  - 确认后调用本地 DAO 入库
- [ ] 设置页 - AI 模型状态查看（模型由管理员在后端配置）

**验收标准**：
- 能通过对话查询资产信息
- 能通过自然语言新增资产
- 能通过截图识别提取数据

## P7：数据同步与导入导出（1-2 周）

### 目标
手机数据备份到服务器，支持加密导出导入。

### 任务清单

**后端 - 数据备份：**
- [ ] 备份上传接口 `POST /api/backup/upload`（接收全量快照 JSON）
- [ ] 备份下载接口 `GET /api/backup/latest`
- [ ] 备份历史列表接口 `GET /api/backup/list`
- [ ] 服务端备份存储（数据库 or 文件系统）

**Android 端 - 同步：**
- [ ] 手动备份按钮（设置页）
- [ ] App 启动时自动备份（后台静默）
- [ ] 备份状态展示（最后备份时间）
- [ ] 从服务端恢复数据（新设备场景）

**Android 端 - 导入导出：**
- [ ] 导出功能
  - 全量资产 + 净值历史 → JSON
  - 全量资产 → Excel (.xlsx)
  - 可选 AES-256 密码加密
  - 调用系统分享
- [ ] 导入功能
  - 选择文件 → 检测格式
  - 加密文件 → 输入密码解密
  - 预览数据 → 确认覆盖/合并
- [ ] Excel 生成库选型（Apache POI Android 版 or 轻量替代）

**验收标准**：
- 备份/恢复流程跑通
- 导出的加密文件能成功导入另一台设备

## 开发节奏建议

> 一个人业余时间开发，最重要的是**保持成就感**。

1. **每个阶段都有可演示的产出** — 不会陷入"做了很久什么都看不到"
2. **P0-P3 是核心骨架** — 完成后就有一个能用的资产记录 App
3. **P4-P6 是差异化功能** — 自动行情 + 报表 + AI 是核心卖点
4. **P7-P8 是完善收尾** — 数据安全感 + 可发布质量
5. **每完成一个阶段就实际使用** — 自己用起来，发现真实痛点

如果某个阶段卡住（如行情爬虫），可以先跳过用手动录入兜底，后面补上。
