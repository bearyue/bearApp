## 一、产品概述

### 1.1 核心定位
**非传统记账工具**——摒弃繁琐的日常流水录入，聚焦于**资产净值（Net Worth）的统一记录、跟踪与智能分析**。
**个人/家庭私有工具**——自部署服务端，仅供个人及家人使用，非商业化多用户产品。

### 1.2 目标用户
- Bear 及家人（有股票、基金、房产等多元资产）
- 有港股/美股持仓的跨市场投资者
- 希望通过 AI 辅助进行资产分析和决策

### 1.3 核心价值
| 痛点 | 解决方案 |
|---|---|
| 资产分散在各平台，无全局视图 | 统一资产仪表盘，多币种本位币折算 |
| 手动更新市值费时费力 | 后端定时爬虫/API 自动更新 + 手动兜底 |
| 资产配置是否合理难以判断 | 完整报表 + AI 分析建议 |
| 数据录入麻烦 | 表单 + AI 对话 + 拍照截图识别，三种方式 |

### 1.4 与竞品差异
| 竞品 | 差异点 |
|---|---|
| 随手记/MoneyWiz | 它们是**流水记账**，我们是**净值追踪**，不记每笔消费 |
| 券商 App | 只看得到单平台持仓，我们做**全平台汇总** |
| Excel 手工表 | 我们有自动行情更新 + AI 分析 + 移动端体验 |

## 二、用户系统

### 2.1 家庭多用户架构
- **轻量多用户**：支持 Bear 及家人各自独立账户、独立数据空间
- 不面向公众开放注册，用户由**管理员（Bear）在后台创建**
- 每个用户只看自己的数据，数据库层面按 `user_id` 隔离
- 不设"家庭汇总"视图（V1），各管各的

### 2.2 认证方式
- **账号密码**：后端管理，管理员创建账号后分配密码
- **生物识别**：本地指纹/面部识别作为快捷解锁（依赖 Android BiometricPrompt API）
- 登录后本地缓存 token，生物识别通过后免密进入

### 2.3 V1 简化策略
- 不做公开注册页面（管理员后台/接口直接创建用户）
- 不做邮箱验证、手机验证码
- 不做忘记密码流程（管理员直接重置）
- 不做 OAuth 第三方登录

## 三、资产建模

### 3.1 资产分类体系

```
资产总览 (Net Worth = 总资产 - 总负债)
├── 流动资产 (Liquid)
│   ├── 现金
│   ├── 活期存款（银行卡余额）
│   └── 货币基金（余额宝、零钱通等）
├── 投资资产 (Investment)
│   ├── 权益类
│   │   ├── A股持仓（股票代码、成本价、数量、当前价）
│   │   ├── 港股持仓（同上 + 港股打新追踪）
│   │   └── 美股持仓（同上）
│   ├── 固收类
│   │   ├── 公募基金（基金代码、份额、净值）
│   │   ├── 银行理财（本金、预期收益率、到期日）
│   │   ├── 定期存款（本金、利率、存期、到期日）
│   │   └── 大额存单（同定期存款）
│   ├── 投顾类
│   │   └── 投顾产品（产品名称、持有金额、持有收益、累计收益；整体管理不拆分底层持仓）
│   └── 其他投资
│       └── 自定义投资项（名称、持有金额、持有收益、累计收益）
├── 受限/保障资产 (Restricted)
│   ├── 公积金（月缴基数、比例、余额；支持按月自动模拟增量）
│   └── 商业保险（险种、累计保费、现金价值）
├── 实体资产 (Physical)
│   ├── 房产（地址、购入价、当前估值）
│   └── 车辆（车型、购入价、当前估值）
└── 负债 (Liabilities)
    ├── 房贷（总额、剩余本金、月供、利率）
    ├── 车贷（同上）
    └── 信用卡（当前未还账单）
```

### 3.2 资产通用字段
每项资产都包含以下基础字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Long | 主键 |
| user_id | Long | 所属用户 |
| category | Enum | 资产大类（Liquid/Investment/Restricted/Physical/Liability） |
| sub_type | String | 子类型（如 stock_a、fund、mortgage 等） |
| name | String | 资产名称（如"招商银行活期"、"贵州茅台"） |
| currency | Enum | 币种（CNY/HKD/USD） |
| amount | Decimal | 当前价值（本币） |
| amount_cny | Decimal | 折算人民币价值（由系统根据汇率计算） |
| cost | Decimal | 成本/投入金额（可选） |
| note | String | 备注 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 最后更新时间 |

### 3.3 多币种支持
- 支持 **CNY / HKD / USD** 三种币种
- 每项资产记录原始币种和原始金额
- 系统自动按**每日汇率**折算为人民币（本位币）
- 汇率来源：后端定时拉取（优先 API，降级为爬虫）
- 汇率缓存在本地，离线时使用最后一次同步的汇率

## 四、核心功能

### 4.1 数据录入（三种方式）

#### 4.1.1 表单录入
- 标准表单界面，按资产类型展示对应字段
- 支持快速新增同类资产（模板复用）
- 证券类资产：输入股票/基金代码后自动填充名称

#### 4.1.2 AI 对话录入
- 用户用自然语言描述，AI 解析后生成结构化数据
- 示例：
  - 用户："买了 1000 股贵州茅台，成本 1850"
  - AI 解析 → `{type: stock_a, code: 600519, name: 贵州茅台, quantity: 1000, cost: 1850}`
  - 展示确认卡片，用户确认后入库
- AI 无法解析时，降级到表单让用户手动补全

#### 4.1.3 拍照/截图识别
- 拍摄或选择券商账单截图、银行 App 截图
- 调用大模型 Vision API 识别文字和数据
- 识别结果展示为可编辑的确认卡片
- 用户确认/修改后入库
- **注意**：图像不上传到后端存储，仅用于实时识别

### 4.2 行情数据更新

#### 4.2.1 自动更新（后端）
- 后端定时任务，工作日收盘后自动更新：
  - A 股：调用行情 API 或爬取行情页面
  - 港股/美股：同上（注意时区差异）
  - 公募基金：爬取天天基金等数据源获取最新净值
  - 汇率：每日更新 CNY/HKD/USD 汇率
- **降级策略**：API 优先 → 爬虫兜底 → 手动录入
- 爬虫工具：V1 待选型（考虑现成开源方案）

#### 4.2.2 手动更新
- 所有资产类型都支持手动修改当前价值
- 适用场景：无 API 覆盖的私募基金、银行理财、房产估值等
- 手动更新时记录更新时间戳

#### 4.2.3 公积金自动模拟
- 用户设定缴存基数、个人比例、单位比例
- 系统按月自动模拟增量（每月 1 号自动增加当月缴存额）
- 用户可随时手动校准实际余额

### 4.3 资产总览仪表盘

#### 4.3.1 首页看板
- **净值大数字**：总资产 - 总负债 = 净值（人民币）
- **资产/负债汇总条**：总资产 | 总负债 | 净值
- **资产分类卡片**：流动/投资/受限/实体/负债各自小计
- **今日涨跌**：投资类资产的日涨跌额和百分比

#### 4.3.2 资产明细列表
- 按大类分组展示
- 每项显示：名称、当前市值（本币）、折合人民币、盈亏
- 支持搜索/筛选

### 4.4 数据可视化（完整报表）

#### 4.4.1 净资产曲线
- 月度/年度净值走势折线图
- 拆解维度：本金投入增量 vs 投资收益增量
- 支持拖拽选择时间范围

#### 4.4.2 资产配置饼图
- 按大类（流动/投资/受限/实体）的占比分布
- 按子类（权益/固收/现金等）的占比分布
- 点击扇区可穿透到明细

#### 4.4.3 风险分析
- 权益类 vs 固收类比例
- 紧急预备金覆盖月数（流动资产 / 月均支出估算）
- 单一资产集中度预警（占比超阈值提醒）

#### 4.4.4 投资绩效
- 单项资产收益率
- 全盘内部收益率 XIRR（基于资金流入流出时间计算）
- 持仓盈亏排行

### 4.5 数据导入导出

#### 4.5.1 导出
- 支持 **Excel (.xlsx)** 和 **JSON** 两种格式
- 导出内容：全量资产快照 + 历史净值记录
- **加密导出**：可选设置密码加密（AES-256），生成加密文件
- 导出文件可通过系统分享发送

#### 4.5.2 导入
- 支持导入本 App 导出的 Excel/JSON 文件
- 加密文件导入时需输入密码解密
- 导入前预览数据，确认后覆盖或合并
- 用途：设备迁移、数据恢复

## 五、AI Agent（Virtual CFO）

### 5.1 架构设计
```
[Android App] → [后端 AI Gateway] → [大模型 API]
                     ↓
              [Function Calling]
              ├── 查询资产数据
              ├── 新增/修改资产
              ├── 生成报表数据
              └── 图像识别解析
```

### 5.2 模型配置
- 后端统一 AI Gateway，对接大模型 API
- **可配置多模型**：支持切换 OpenAI GPT / Claude / 国产模型（通义千问/文心一言等）
- 配置项：API Key、模型名称、Base URL、温度等参数
- API Key 统一存储在后端，客户端无需关心密钥管理

### 5.3 功能场景

#### 5.3.1 自然语言查询
- "我现在总资产多少？"
- "港股持仓占总资产多少比例？"
- "我今年投资赚了多少？"
- 实现：Agent 通过 Function Calling 调用后端数据查询接口，组装上下文后回答

#### 5.3.2 AI 辅助录入
- 自然语言 → 结构化数据解析（见 4.1.2）
- 图像识别 → 数据提取（见 4.1.3）

#### 5.3.3 智能分析建议
- 资产配置风险提醒："单只科技股持仓占比超过 40%，建议关注分散风险"
- 趋势分析："近 3 个月净值增长 X%，主要来自权益类资产增值"
- **V1 范围**：基于当前快照数据的分析，不做复杂预测模型

### 5.4 隐私策略
- 发送给大模型的数据为**聚合统计数据**，不发送具体账户信息
- 对话历史存储在本地和自建后端，不经过第三方（自建服务器自己掌控）
- 图像识别调用 Vision API 后不保存原图

## 六、技术架构

### 6.1 整体架构
```
┌─────────────────────────────────────────────┐
│                Android App                   │
│  Jetpack Compose + Room + Retrofit + Hilt   │
│  本地数据库（Room）为数据主源               │
└─────────────────┬───────────────────────────┘
                  │ HTTPS
┌─────────────────▼───────────────────────────┐
│     Spring Boot 后端 (自部署·家庭私有)       │
│  ├── 用户认证（JWT，管理员创建账号）         │
│  ├── 数据备份接口（手机→服务器单向）        │
│  ├── AI Gateway（代理大模型请求）           │
│  ├── 行情数据服务（API + 爬虫）            │
│  └── 汇率同步服务                           │
│  数据库：PostgreSQL                         │
│  ⚠ 仅家庭内部使用，不对外开放               │
└─────────────────────────────────────────────┘
```

### 6.2 客户端技术栈
| 组件 | 技术选型 | 说明 |
|---|---|---|
| UI 框架 | Jetpack Compose | 声明式 UI，Material Design 3 |
| 本地数据库 | Room | 资产数据本地持久化 |
| 网络请求 | Retrofit + OkHttp | 与后端通信 |
| 依赖注入 | Hilt | Android 官方推荐 DI |
| 图表库 | MPAndroidChart 或 Vico | 曲线图、饼图等 |
| 图片处理 | Coil | 图片加载 |
| 架构 | MVVM + Clean Architecture | Repository 模式 |
| 导航 | Navigation Compose | 单 Activity 多 Compose Screen |
| UI 风格 | 工具类简洁风 | 参考 Notion/滴答清单风格 |

### 6.3 后端技术栈
| 组件 | 技术选型 | 说明 |
|---|---|---|
| 框架 | Spring Boot 3.x | Java/Kotlin |
| ORM | MyBatis-Plus | 数据库操作 |
| 认证 | Spring Security + JWT | 无状态认证 |
| 数据库 | PostgreSQL 16 | 用户数据 + 备份数据 |
| 定时任务 | Spring Scheduler | 行情/汇率定时拉取 |
| AI 代理 | RestTemplate / WebClient | 转发大模型 API 请求 |
| 部署 | Docker + 自有服务器 | 家庭内网 / 个人 VPS |

### 6.4 数据同步策略
- **方向**：手机 → 服务器（单向备份）
- **时机**：用户手动触发 / App 启动时自动备份
- **机制**：
  - 客户端记录每条数据的 `updated_at` 时间戳
  - 备份时将本地所有变更推送到服务器
  - 服务器存储最新快照，不做冲突合并
- **恢复**：新设备安装后，从服务器拉取最近备份恢复到本地

### 6.5 V1 不做的事（明确排除）
- ❌ 公开注册 / 商业化多用户（非目标场景）
- ❌ 数据端到端加密（V2）
- ❌ 双向同步 / 多设备实时同步（V2）
- ❌ SQLCipher 本地加密数据库（V2）
- ❌ 隐私保险箱功能（V2）
- ❌ 数据脱敏视图（V2）
- ❌ 健康管理模块（V2+）
- ❌ 纪念日模块（V2+）
- ❌ iOS 客户端（V2+）
- ❌ OAuth 第三方登录
- ❌ 邮箱/短信验证

## 七、数据模型（核心 ER 概要）

### 7.1 用户表 (user)
```sql
user_id       BIGINT PK AUTO_INCREMENT
username      VARCHAR(50) UNIQUE NOT NULL
password_hash VARCHAR(255) NOT NULL
nickname      VARCHAR(50)
created_at    DATETIME
updated_at    DATETIME
-- 管理员通过后台接口创建用户，不开放公开注册
```

### 7.2 资产表 (asset)
```sql
asset_id      BIGINT PK AUTO_INCREMENT
user_id       BIGINT FK → user
category      ENUM('LIQUID','INVESTMENT','RESTRICTED','PHYSICAL','LIABILITY')
sub_type      VARCHAR(50)       -- stock_a, stock_hk, stock_us, fund, advisor(投顾), deposit, mortgage...
name          VARCHAR(200)
currency      ENUM('CNY','HKD','USD')
amount        DECIMAL(18,4)     -- 当前价值（原币）
amount_cny    DECIMAL(18,4)     -- 折合人民币
cost          DECIMAL(18,4)     -- 成本
quantity      DECIMAL(18,4)     -- 数量/份额（证券类）
code          VARCHAR(20)       -- 证券/基金代码
extra_json    JSON              -- 扩展字段（利率、到期日、月供等）
note          TEXT
created_at    DATETIME
updated_at    DATETIME
```

### 7.3 净值快照表 (net_worth_snapshot)
```sql
snapshot_id   BIGINT PK AUTO_INCREMENT
user_id       BIGINT FK → user
snapshot_date DATE
total_asset   DECIMAL(18,4)
total_liability DECIMAL(18,4)
net_worth     DECIMAL(18,4)
breakdown_json JSON              -- 各类资产小计明细
created_at    DATETIME
```

### 7.4 汇率表 (exchange_rate)
```sql
rate_id       BIGINT PK AUTO_INCREMENT
from_currency ENUM('HKD','USD')
to_currency   ENUM('CNY') DEFAULT 'CNY'
rate          DECIMAL(12,6)
rate_date     DATE
created_at    DATETIME
```

### 7.5 资产变动日志 (asset_change_log)
```sql
log_id        BIGINT PK AUTO_INCREMENT
asset_id      BIGINT FK → asset
user_id       BIGINT FK → user
change_type   ENUM('CREATE','UPDATE','DELETE','PRICE_UPDATE')
old_value     JSON
new_value     JSON
changed_at    DATETIME
```

### 7.6 AI 对话记录 (ai_chat_history)
```sql
chat_id       BIGINT PK AUTO_INCREMENT
user_id       BIGINT FK → user
role          ENUM('USER','ASSISTANT','SYSTEM')
content       TEXT
model_used    VARCHAR(50)
created_at    DATETIME
```
